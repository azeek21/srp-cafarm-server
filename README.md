# Cattle Management System — Server

Backend API for a single-farmer livestock management system. Manages cattle records (CRUD with
soft-delete), filtered/sorted/paginated browsing, and herd analytics. The React client lives in a
separate repo.

**Stack:** Python ≥ 3.11 · FastAPI · SQLModel · sync `psycopg` · PostgreSQL · `uv` + `ruff`.

> Architecture, data model, and design decisions: see `agent-docs/plan_v1.md`.

## Prerequisites

- [`uv`](https://docs.astral.sh/uv/) (provisions Python ≥ 3.11)
- PostgreSQL running locally

## Setup

```bash
createdb farm_local                     # app database
cp .env.example .env                    # set DATABASE_URL + CORS_ORIGINS
uv sync                                 # install deps (incl. dev group)
```

`.env`:

```ini
DATABASE_URL=postgresql+psycopg://USER:PASSWORD@localhost:5432/farm_local
CORS_ORIGINS=http://localhost:5173,http://localhost:3000
```

## Run

```bash
uv run uvicorn app.main:app --reload    # http://localhost:8000
```

Schema is managed by **Alembic migrations**. On startup the lifespan runs
`alembic upgrade head` (via `init_db`), so a fresh DB is brought to the latest schema
automatically. See **Migrations** below to create/apply them manually.

- Swagger UI: `http://localhost:8000/docs`
- Health: `GET http://localhost:8000/health` → `{"status":"ok"}`

Seed ~40 sample cattle:

```bash
uv run python -m app.seed
```

## Test

Integration tests run against a **separate** database (`farm_local_test`, same creds, different
db name) using FastAPI `TestClient`. Tables are created/dropped per session; each test starts from
a clean table.

```bash
createdb farm_local_test                # one-time
uv run pytest                           # run suite
uv run pytest tests/test_animals_api.py # single file
uv run pytest -k soft_delete -v         # by keyword
```

The test DB URL is set in `tests/conftest.py` (overrides `DATABASE_URL` before app import).
The test schema is built/torn down per session by running migrations (`upgrade head` /
`downgrade base`), matching the production code path.

## Migrations

Alembic config: `alembic.ini` + `migrations/` (env reads `DATABASE_URL` from settings).

```bash
uv run alembic upgrade head                       # apply pending migrations
uv run alembic revision --autogenerate -m "msg"   # generate from model changes
uv run alembic downgrade -1                        # roll back one
uv run alembic current                            # show applied revision
```

Autogenerated scripts are a **starting point** — review them. Notably, partial indexes and
native PG enum value changes (`ALTER TYPE`) are not fully detected and may need hand-editing.

## API

Base prefixes: `/api/animals`, `/api/analytics`.

### Animals
- `POST /api/animals/` — create. **409** on duplicate `(type, tag)` among live animals.
- `GET /api/animals/` — list → `{ items, total, page, page_size }`. Query params:
  `type`, `search` (ILIKE over tag/name/breed), `breed`, `gender`, `status`,
  `born_after`, `born_before`, `sort_by` (`tag|breed|date_of_birth|status|created_at`),
  `sort_order` (`asc|desc`), `page` (≥1), `page_size` (1–100). `total` counts the full filtered
  set pre-pagination.
- `GET /api/animals/{id}` — fetch one. **404** if missing/soft-deleted.
- `PATCH /api/animals/{id}` — partial update. Touches `updated_at`. **404** / **409**.
- `DELETE /api/animals/{id}` — soft delete (sets `deleted_at`). **204** / **404**.

### Analytics
- `GET /api/analytics/summary?type=cattle` → `total`, `active_total`, `by_status`, `by_breed`,
  `by_gender`, `age_distribution` (calf `<1y`, young `1–2y`, adult `>2y`), `average_age_years`.

`age_years` is derived from `date_of_birth` at read time. Deletes are soft — deleted rows are
excluded from every read, and a tag can be reused after its animal is deleted.
